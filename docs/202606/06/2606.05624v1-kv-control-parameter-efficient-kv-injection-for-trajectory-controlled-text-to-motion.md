## 摘要
文本条件下的3D人体运动模型现在可以根据提示合成合理的运动，但实际的动画和具身智能体工作流程很少止步于文本：角色可能需要遵循草绘的根路径，击中末端执行器目标，或满足多关节轨迹，同时仍保留语言描述的步态、风格和意图。这暴露了一个控制权衡问题。轨迹控制器应该精确而不覆盖预训练的文本条件运动先验，但现有的解决方案要么复制生成器的大部分以重新获得每层控制访问，要么将大部分成本转移到测试时优化。我们引入KV-Control，一个用于冻结掩码文本到运动变换器的紧凑注意力侧控制接口。关键思想是将几何约束作为记忆在自注意力内部提供，而不是通过全局姿态标记注入或仅在输出侧强制执行。为支持此接口，我们共同设计了一个部分标记化的运动基座和控制器：\textbf{PartVQ}学习解剖学对齐的部分码本，T-Concat将每个帧-部分标记暴露为可寻址注意力的站点，KV-Control在每个自注意力层注入控制条件的键/值记忆，同时保留预训练的查询流、文本交叉注意力、FFN和所有骨干权重。由此产生的适配器仅在共享轨迹编码器之上添加可训练的注入参数，但在继承的精炼协议下以亚厘米精度跟踪根和多关节约束，同时保留文本条件下的运动质量。KV-Control将轨迹条件重新定义为轻量级记忆检索，为文本到运动生成提供了小型、精确且透明的控制接口。

## Abstract
Text-conditioned 3D human motion models now synthesize plausible motions from prompts, but practical animation and embodied-agent workflows rarely stop at text: a character may need to follow a sketched root path, hit an end-effector target, or satisfy a multi-joint trajectory while still preserving the gait, style, and intent described by language. This exposes a control trade-off. A trajectory controller should be precise without overwriting the pretrained text-conditioned motion prior, yet existing solutions either duplicate large portions of the generator to regain per-layer control access or move much of the cost to test-time optimization. We introduce KV-Control, a compact attention-side control interface for frozen masked text-to-motion transformers. The key idea is to make geometric constraints available as memory inside self-attention rather than injecting them through a global pose token or enforcing them only at the output side. To support this interface, we co-design a part-tokenized motion substrate and controller: \textbf{PartVQ} learns anatomy-aligned part codebooks, T-Concat exposes each frame--part token as an attention-addressable site, and KV-Control injects control-conditioned key/value memories at every self-attention layer while preserving the pretrained query stream, text cross-attention, FFN, and all backbone weights. The resulting adapter adds only trainable injection parameters atop a shared trajectory encoder, yet tracks root and multi-joint constraints with sub-centimeter accuracy under the inherited refinement protocol while retaining text-conditioned motion quality. KV-Control reframes trajectory conditioning as lightweight memory retrieval, providing a small, precise, and transparent control interface for text-to-motion generation.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：文本条件3D人体运动生成模型已能合成合理运动，但实际应用（动画、具身智能体）需要精确的轨迹控制（如根路径、末端执行器位置、多关节轨迹），同时保留文本描述的步态、风格和意图。
- **现有局限**：现有方法存在控制权衡——要么复制生成器大部分（如ControlNet风格）以获得每层注意力访问，但参数巨大；要么通过测试时优化（如TLControl、DNO）避免训练参数，但增加推理计算成本。两者均未能以极小参数代价实现逐层注意力侧控制。
- **本文目标**：提出一个参数高效、精确的轨迹控制接口，在冻结的文本到运动Transformer骨干上实现亚厘米级轨迹跟踪，而几乎不影响生成质量。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- 将轨迹控制重新定义为**注意力记忆检索**：在冻结自注意力层的键/值侧（K/V）注入控制条件记忆，而不是改变查询（Q）流或全局姿态标记。
- 联合设计三个组件：PartVQ（解剖学部件分词）、T-Concat（部件词元展开）、KV-Control（键/值注入），实现每个解剖部件-帧词元作为可寻址注意力站点。

### 关键技术细节

#### 2.1 PartVQ：解剖感知部件分词
- 基于数据驱动的聚类（滞后互相关+运动链完整性后处理）将人体关节分为6个解剖部件。
- 每个部件学习独立的VQ-VAE码本（128维，128个码字），编码器为1D卷积，解码器共享。
- 损失：smooth-L1重建 + VQ承诺 + 关节位置smooth-L1。

#### 2.2 T-Concat骨干网络
- 将6个部件码本沿序列轴展开（而非通道拼接），得到S = Ttok × Q = 294个词元（原49帧×6部件）。
- 每个词元位置添加位置编码（时间嵌入+部件索引嵌入），保持每个部件词元身份可寻址。
- 文本条件：使用冻结CLIP文本编码器 + 可训练的小型Transformer适配器，在每层自注意力后插入门控文本交叉注意力。

#### 2.3 KV-Control：键/值注入机制
- **控制编码器**：在每个采样步骤，通过冻结PartVQ解码器获得当前预测姿态，计算残差（目标 - 预测），结合目标位置和关节掩码，经1D卷积下采样到Ttok个dmodel维特征fctrl。
- **逐层注入**：每层自注意力中，fctrl通过零初始化的低秩下投影（rank r=64）和两个并行上投影生成控制列的K/V（C_K, C_V），与原始词元序列拼接；查询Q保持不变；添加可学习注意力偏置b_i（初始化为强负数-5），使控制器起始为近似恒等映射。
- **设计选择**：选择K/V侧而非Q侧——保护Q序列不受干扰，从而保留预制文本交叉注意力。

#### 2.4 训练与推理
- **冻结骨干**：PartVQ、CLIP适配器、T-Concat骨干完全冻结，仅训练控制编码器、逐层K/V投影和偏置。
- **损失函数**：L_KV = λ_CE L_CE + λ_traj L_traj，其中L_CE为掩码词元交叉熵，L_traj为FK监督（通过软码本松弛可微分）。

## 6. 主要结论与发现

- **参数效率**：KV-Control仅需1.5M机制参数（10.5M含编码器）即可达到0.40cm（单关节骨盆M3）和0.71cm（多关节M3）的位置误差，而ControlNet分支需39.3M才能达到1.24cm（M2），参数效率提升约26×。
- **精确控制**：在最强精炼（M3）下，单关节误差0.40cm，多关节0.71cm，达到与全复制方法相当甚至更好的精度。
- **质量保持**：FID和Top-3与无控制基线几乎无退化（FID ~0.065单关节、~0.054多关节；Top-3 ~0.80）。
- **布局关键**：T-Concat（部件词元沿序列展开）相比C-Concat（通道拼接）带来约2倍的位置误差降低，证明解剖可寻址布局对参数高效控制至关重要。
- **阶段演化**：M0（纯前馈）误差较大（~30cm），M1（Stage1）降至~1cm，M2（+Stage2）进一步降至~0.5cm，M3（动态Stage1）达到最佳~0.4cm。精炼仅基线（无K/V适配器）虽位置误差近零，但FID塌陷至~104，证实适配器提供结构条件使精炼保持在运动流形内。

## 7. 优点

1. **参数极致高效**：极少量可训练参数（1.5M）即实现精确轨迹跟踪，远低于同类ControlNet风格方法。
2. **保留骨干质量**：冻结骨干完全不变，生成质量无退化，且可插拔。
3. **设计巧妙**：将轨迹控制重新定义为记忆检索，选择K/V侧注入而非Q侧，保护文本条件；近恒等初始化使控制器平滑启动。
4. **系统消融**：细致分析了骨干布局、控制机制、精炼配置的影响，验证了每个设计的必要性。
5. **定性验证充分**：展示了OOD字母路径、多关节约束等复杂场景，证明了泛化能力。

## 8. 不足与局限

1. **数据集单一**：仅在HumanML3D上评估，未在KIT-ML、AMASS等数据集验证泛化性。
2. **对比不完全公平**：部分对比方法（如TLControl）使用不同协议（直接引用原论文数据），本文未在自己设置下复现它们；ControlNet分支仅在同一骨干下复现，未与原始OmniControl/MaskControl在完全一致环境比。
