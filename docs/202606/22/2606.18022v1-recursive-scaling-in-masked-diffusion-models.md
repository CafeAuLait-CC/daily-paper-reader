## 摘要
掩码扩散模型（MDMs）最近已成为序列生成的一种有前景的范式。传统上，扩展MDMs通过增加参数数量或去噪步骤数来实现。我们引入了递归掩码扩散模型（R-MDMs），通过在每次扩散步骤中重复应用相同的去噪变换器，将递归深度作为第三个扩展轴。递归使得通过参数复用能够迭代细化输出，从而在不增加参数数量的情况下增加有效模型深度。在包括数独和倒数游戏在内的结构化生成任务中，我们展示了R-MDMs实现了显著提升的参数效率：具有$L$次递归迭代的模型通常能与参数数量约为$L$倍的非递归基线模型性能相当。此外，递归细化可以部分替代额外的去噪步骤，使得递归模型在推理时使用更少的前向传播即可达到相同的生成质量。这些结果表明，递归深度是MDMs的一种实用的扩展机制，既提高了参数效率，也优化了测试时计算资源的分配。

## Abstract
Masked diffusion models (MDMs) have recently emerged as a promising paradigm for sequence generation. Scaling MDMs is conventionally achieved by increasing the parameter count or the number of denoising steps. We introduce Recursive Masked Diffusion Models (R-MDMs), which add recursive depth as a third scaling axis by repeatedly applying the same denoising transformer within each diffusion step. Recursion enables iterative refinement of the output through parameter reuse, increasing effective model depth without increasing parameter count. Across structured generation tasks, including Sudoku and Countdown, we show that R-MDMs achieve substantially improved parameter efficiency: a model with $L$ recursive iterations often matches the performance of non-recursive baselines with roughly $L\times$ more parameters. Moreover, recursive refinement can partially substitute for additional denoising steps, allowing recursive models to reach the same generation quality with fewer forward passes at inference time. These results suggest that recursive depth is a practically useful scaling mechanism for MDMs, improving both parameter efficiency and the allocation of test-time compute.

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- 在每个去噪步内，将同一 K 层 transformer 块 `fθ` 循环应用 L 次（参数共享），实现有效深度 K×L，参数数等同于 K 层模型。
- 利用 MDM 的双向注意力特性，使每次递归都能全局传播约束，比自回归递归更高效。

### 关键技术细节
- **架构**：给定噪声输入 `xt`，首先进行嵌入和归一化得到 `h^(0)`；然后循环 L 次：`h^(ℓ) = fθ(h^(ℓ-1), ℓ, L)`；最后通过共享头部 `Wθ` 输出 logits `ŷ^(ℓ)`。
- **递归步嵌入**：将归一化递归进度 `s_ℓ = (ℓ-1)/(L-1)` 通过两层 MLP 映射为 d 维向量，加在隐藏状态前，使模型感知当前循环位置。
- **训练目标**：主要采用 **All-steps 损失**（`Lall`），即对所有 L 个循环的输出都计算交叉熵并取平均，提供密集梯度。其他变体包括 Final、Weighted、Truncated。
- **递归步选择策略**：训练时可固定递归步数，也可随机采样，或采用 **反向课程（Schedule 10→1）**——先高后低，在实验中被证明最优。
- **测试时灵活调节**：训练后的模型可自由选择推理时的递归步数 `Ls`，实现连续的 compute-quality trade-off。

## 6. 主要结论与发现

1. **递归深度可替代参数扩展（RQ1）**：在 Sudoku 和 Countdown 上，`(K⊗L)` 模型匹配 `(KL⊗1)` 模型性能，节省约 L 倍参数。例如 6层递归5次（10.6M）在 Sudoku 9×9 上达到 30层基线（53.1M）的性能。
2. **递归可减少去噪步数（RQ2）**：在 Sudoku 上，`Ls=3, T=5`（总前向 15 次）达到 95.6% VPR，而非递归基线需 `T=40` 次，计算减少 2.7 倍。递归还提高了性能上限（>98% vs 基线饱和~96%）。
3. **任务依赖性**：结构化任务（Sudoku, Countdown）收益显著；无结构字符建模（Text8）上递归效果不佳甚至更差，但生成文本更连贯。
4. **反向课程调度（Schedule 10→1）** 和 **All-steps 损失** 是较优的训练配置。
5. **递归与去噪步可互换**，测试时可灵活搭配，实现高效的 compute-quality 权衡。

## 7. 优点

- **方法新颖**：将递归深度作为 MDM 的独立扩展轴，区别于传统参数/步数扩展。
- **全面的实验设计**：覆盖多个结构化难度、两种对比基线（iso-parameter / iso-FLOP）、消融完整、提供了详细的延迟和吞吐量数据。
- **实用性强**：递归模型在参数效率和推理效率上同时优于非递归基线，特别适用于资源受限场景。
- **开源友好**：明确使用 LLM 辅助写代码但作者审核，透明度高。
- **技术深度**：深入分析了递归与去噪步的互换性，给出了 Pareto 前沿和 compute-efficient 推理曲线。

## 8. 不足与局限

- **实验规模有限**：最大模型仅约 50M 参数，序列最长 625 tokens，未在大规模预训练场景下验证。作者也承认这是主要局限。
- **训练开销**：All-steps 损失需存储所有循环的激活值，导致 L× 的内存和计算开销（虽不增加参数量）。
- **对训练步数的依赖**：模型表示紧密校准到训练时的递归步数，若推理时步数远小于训练值，性能急剧下降（虽然可通过反向课程缓解）。
- **无结构任务不适用**：Text8 上递归未改善 NLL，且 Likelihood 指标可能掩盖实际质量，作者未进行人工评估或更丰富的自动指标。
- **缺乏机制分析**：未深入解释递归块内部到底学会了什么迭代运算（如是否近似解约束满足算法），仅根据性能推断。
