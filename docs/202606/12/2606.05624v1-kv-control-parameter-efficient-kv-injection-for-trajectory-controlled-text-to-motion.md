## 摘要
基于文本条件的3D人体运动模型现在可以从提示中合成合理的运动，但实际的动画和具身智能体工作流程很少停留在文本上：角色可能需要遵循手绘的根路径、达到末端执行器目标或满足多关节轨迹，同时仍然保持语言描述的步态、风格和意图。这暴露了控制上的权衡。轨迹控制器应该精确，而不会覆盖预训练的文本条件运动先验，但现有解决方案要么复制生成器的大部分以重新获得每层控制访问，要么将大部分成本转移到测试时优化。我们引入了KV-Control，这是一种针对冻结的掩码文本到运动变换器的紧凑注意力侧控制接口。关键思想是使几何约束作为自注意力内部的记忆可用，而不是通过全局姿态令牌注入它们或仅在输出侧强制执行它们。为了支持这一接口，我们共同设计了一个部分令牌化的运动基元和控制器：\textbf{PartVQ}学习解剖对齐的部分码本，T-Concat将每帧-部分令牌暴露为可寻址的注意力位置，KV-Control在每个自注意力层注入控制条件的键/值记忆，同时保留预训练的查询流、文本交叉注意力、FFN和所有骨干权重。生成的适配器仅在共享的轨迹编码器上添加可训练的注入参数，但在继承的细化协议下以亚厘米精度跟踪根和多关节约束，同时保持文本条件的运动质量。KV-Control将轨迹条件重构为轻量级记忆检索，为文本到运动生成提供了一个小巧、精确且透明的控制接口。

## Abstract
Text-conditioned 3D human motion models now synthesize plausible motions from prompts, but practical animation and embodied-agent workflows rarely stop at text: a character may need to follow a sketched root path, hit an end-effector target, or satisfy a multi-joint trajectory while still preserving the gait, style, and intent described by language. This exposes a control trade-off. A trajectory controller should be precise without overwriting the pretrained text-conditioned motion prior, yet existing solutions either duplicate large portions of the generator to regain per-layer control access or move much of the cost to test-time optimization. We introduce KV-Control, a compact attention-side control interface for frozen masked text-to-motion transformers. The key idea is to make geometric constraints available as memory inside self-attention rather than injecting them through a global pose token or enforcing them only at the output side. To support this interface, we co-design a part-tokenized motion substrate and controller: \textbf{PartVQ} learns anatomy-aligned part codebooks, T-Concat exposes each frame--part token as an attention-addressable site, and KV-Control injects control-conditioned key/value memories at every self-attention layer while preserving the pretrained query stream, text cross-attention, FFN, and all backbone weights. The resulting adapter adds only trainable injection parameters atop a shared trajectory encoder, yet tracks root and multi-joint constraints with sub-centimeter accuracy under the inherited refinement protocol while retaining text-conditioned motion quality. KV-Control reframes trajectory conditioning as lightweight memory retrieval, providing a small, precise, and transparent control interface for text-to-motion generation.

## 1. 论文的核心问题与整体含义

**研究动机与背景**：文本到3D人体运动生成模型已能生成语义合理的运动，但实际动画和具身智能体工作流中，角色常需遵循手绘根轨迹、达到末端执行器目标或满足多关节轨迹，同时保持文本描述的步态、风格和意图。现有轨迹控制方法存在权衡：一类复制生成器大部分层（如ControlNet风格）以恢复每层自注意力访问，但参数量大；另一类将成本转移到测试时优化（如DNO、TLControl），增加推理计算。两者都未能在参数效率和推理成本之间取得良好平衡。

**核心问题**：如何在不覆盖预训练文本条件运动先验的前提下，实现精确的轨迹控制，且控制适配器的可训练参数和架构足迹远小于骨干网络。

## 2. 论文提出的方法论

**核心思想**：将几何约束作为自注意力内部的记忆（K/V注入），而不是通过全局姿态令牌或仅在输出侧强制执行。为此，共同设计了三个组件：PartVQ、T-Concat和KV-Control。

- **PartVQ**：数据驱动的解剖感知部件分词。通过层级聚类将22个关节划分为6个身体部位（如盆腔、头、左右臂、左右腿），每个部件学习独立的码本（codebook size 128, code dim 128）。使用1D卷积编码器、每部件EMA码本量化和共享解码器，训练结合平滑L1重建、VQ承诺损失和关节位置平滑L1损失。

- **T-Concat骨干**：将PartVQ的6个码本沿序列轴展开，每个（帧, 部件）对成为一个独立的注意力位置（S = 294 tokens），而非通道拼接。位置编码为时间嵌入+部件索引嵌入之和。文本路径使用冻结CLIP + 可训练小Transformer编码器，每层自注意力后插入门控文本交叉注意力。骨干为后归一化Transformer，掩码令牌预测训练，额外使用2D混合掩码正则化。

- **KV-Control**：在每个自注意力层，将控制信号作为额外的K/V伪令牌注入。具体：一个共享的1D卷积轨迹编码器处理当前状态预测与目标的残差及目标位置，输出控制特征 \(c_s\)。每层通过零初始化的低秩下投影 \(W_{down}^i\) 和两个平行上投影 \(W_K^i, W_V^i\) 生成控制键 \(CK_i\) 和值 \(CV_i\)。输入序列扩展为 \([X_i; CK_i]\) 和 \([X_i; CV_i]\)，查询 \(Q_i\) 不变。注意力偏置 \(B_i\) 在控制令牌列初始化为强负值（-5），使控制器初始接近恒等映射。训练目标为掩码交叉熵 + 轨迹L1损失（通过软码本松弛可微）。

**算法流程简述**：
1. 预训练PartVQ和T-Concat骨干（冻结）。
2. 控制适配阶段：同骨干冻结，训练轨迹编码器、每层低秩投影和偏置。输入为目标轨迹和当前token状态，通过冻结PartVQ解码器得到预测位置，计算残差，控制编码器输出，每层注入K/V，最终输出运动令牌。
3. 推理时可选测试时优化（M0/M1/M2/M3四档操作点）。

## 6. 论文的主要结论与发现

- KV-Control在冻结的PartVQ+T-Concat骨干上，仅用1.5M K/V注入参数（完整侧10.5M），即实现盆腔轨迹误差0.40 cm（M3）、多关节0.71 cm，同时保持文本条件运动质量（FID ~0.065, Top-3 ~0.799）。
- 与ControlNet副本分支（39.3M机制参数）在M2协议下相比，KV-Control误差0.51 cm vs. 1.24 cm，参数减少约26×。
- 骨干布局至关重要：T-Concat展开部件token优于通道拼接C-Concat，因为每个（帧, 部件）作为独立注意力位置，便于低秩K/V残差选择性调制。
- 适配器不是细化的替代：纯细化（无K/V）导致FID灾难性退化（~104），适配器提供了结构条件使细化保持在文本条件运动流形内。

## 7. 优点

- **参数高效**：仅注入少量K/V伪令牌，不修改骨干权重，远少于ControlNet风格副本。
- **注意力侧注入**：选择K/V侧而非Q侧，避免干扰文本交叉注意力，保护预训练文本条件能力。
- **部件级别可寻址**：PartVQ+T-Concat将解剖部件作为独立token，使控制信号能精准作用于相关部位。
- **近恒等初始化**：低秩投影零初始化、偏置强负值，使控制器从零开始学习，无需手动调度。
- **灵活的测试时细化**：提供M0-M3四档操作点，从纯前馈到强细化，用户可按精度/计算需求选择。
- **定性结果丰富**：展示了多种OOD轨迹和关节约束的合理运动，证明泛化性。

## 8. 不足与局限

- **转移性有限**：仅验证在HumanML3D数据集和掩码Transformer骨干上，未在其他架构（扩散、自回归）或其他数据集上验证。
- **密集约束压力**：密集每帧接触和多关节约束下，Stage-2细化可能过度固定，导致FID下降（如脚部FID > 0.1），需更优正则化。
- **单骨架依赖**：PartVQ分区基于单一人体骨架，迁移到不同形态（如动物、四足）需重新设计分区。
- **算力信息缺失**：未报告训练时间、GPU数量等，不利于复现和资源评估。
- **端到端效率**：虽然参数少，但推理时若启用M2/M3细化仍需10-15秒/样本，不适合实时应用。
- **偏差风险**：使用MaskControl协议和特定种子，可能需要更多随机性分析。
