## 摘要
文本条件的三维人体动作模型现在可以从提示中合成合理的动作，但实际的动画和具身智能体工作流程很少止于文本：角色可能需要遵循草图化的根路径、到达末端执行器目标或满足多关节轨迹，同时仍保持语言所描述的步态、风格和意图。这暴露了一个控制权衡。轨迹控制器应精确而不覆盖预训练的文本条件动作先验，然而现有解决方案要么复制生成器的大部分以重新获得每层控制访问，要么将大部分成本转移到测试时优化。我们引入了KV-Control，一种用于冻结掩码文本到动作变压器的紧凑注意力侧控制接口。关键思想是将几何约束作为记忆在自注意力内部可用，而不是通过全局姿态令牌注入或仅在输出端强制执行。为了支持这一接口，我们共同设计了一个部分令牌化的动作基元和控制器：\textbf{PartVQ}学习解剖对齐的部分码本，T-Concat将每个帧-部分令牌暴露为注意力可寻址站点，KV-Control在每个自注意力层注入控制条件的键/值记忆，同时保留预训练的查询流、文本交叉注意力、FFN和所有骨干权重。生成的适配器仅在共享轨迹编码器上添加可训练的注入参数，但在继承的细化协议下以亚厘米精度跟踪根和多关节约束，同时保持文本条件的动作质量。KV-Control将轨迹条件重构为轻量级记忆检索，为文本到动作生成提供了小巧、精确且透明的控制接口。

## Abstract
Text-conditioned 3D human motion models now synthesize plausible motions from prompts, but practical animation and embodied-agent workflows rarely stop at text: a character may need to follow a sketched root path, hit an end-effector target, or satisfy a multi-joint trajectory while still preserving the gait, style, and intent described by language. This exposes a control trade-off. A trajectory controller should be precise without overwriting the pretrained text-conditioned motion prior, yet existing solutions either duplicate large portions of the generator to regain per-layer control access or move much of the cost to test-time optimization. We introduce KV-Control, a compact attention-side control interface for frozen masked text-to-motion transformers. The key idea is to make geometric constraints available as memory inside self-attention rather than injecting them through a global pose token or enforcing them only at the output side. To support this interface, we co-design a part-tokenized motion substrate and controller: \textbf{PartVQ} learns anatomy-aligned part codebooks, T-Concat exposes each frame--part token as an attention-addressable site, and KV-Control injects control-conditioned key/value memories at every self-attention layer while preserving the pretrained query stream, text cross-attention, FFN, and all backbone weights. The resulting adapter adds only trainable injection parameters atop a shared trajectory encoder, yet tracks root and multi-joint constraints with sub-centimeter accuracy under the inherited refinement protocol while retaining text-conditioned motion quality. KV-Control reframes trajectory conditioning as lightweight memory retrieval, providing a small, precise, and transparent control interface for text-to-motion generation.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：文本驱动的3D人体动作生成模型已能根据文本提示合成合理动作，但实际动画和具身智能体工作流中，角色常需跟随特定轨迹（如根路径、末端执行器位置）同时保留文本指定的步态、风格和意图。现有轨迹控制器面临两难：要么复制生成器大量层以获得每层控制权（如ControlNet风格，参数开销大），要么依赖测试时优化（如TLControl、DNO，推理计算成本高）。
- **意义**：需要一种轻量、精确且透明的控制接口，在不破坏预训练文本条件先验的前提下实现轨迹约束注入。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- **将几何约束作为自注意力中的键/值记忆注入**，而非通过全局姿态令牌或仅在输出端施加约束。在冻结的掩码文本到运动Transformer中，每层自注意力附加控制条件键/值对，保留查询流、文本交叉注意力、FFN和骨干权重不变。

### 三大组件
1. **PartVQ（解剖感知部件令牌化）**  
   - 数据驱动地将22个人体关节划分为6个身体部分（通过滞后互相关聚类和运动链完整性后处理）。  
   - 每个部分独立学习码本（128大小、128维度），通过1D卷积编码器和共享解码器训练VQ-VAE。  
   - 输出：每个部分每帧一个令牌。
2. **T-Concat（令牌序列级联骨干）**  
   - 将PartVQ的6个码本沿序列轴展开，得到 `S = T_tok * Q` 个令牌（例如 `49帧*6=294`），每个(帧, 部分)对是一个独立的注意力可寻址位置，保留逐位置文本交叉注意力。  
   - 位置编码：时间嵌入+量化器嵌入相加。  
   - 骨干：20层后归一化Transformer（`d_model=384`，8头），带门控文本交叉注意力，训练掩码令牌预测。
3. **KV-Control（键/值注入机制）**  
   - 每层自注意力：从共享轨迹编码器提取控制特征 `f_ctrl`，通过零初始化的低秩下投影（`W_down_i`）和两个并行上投影（`W_K_i`, `W_V_i`）生成控制条件键值对 `CK_i`、`CV_i`，附加到原始键/值序列上。  
   - 查询流保持不变，添加可学习列注意力偏置（初始化为强负值，使控制器初始近似恒等映射）。  
   - 动机：键/值侧干预不影响查询流，从而不扰动文本交叉注意力；初始化接近恒等，可平滑学习。
- **训练目标**：`L_KV = λ_CE L_CE + λ_traj L_traj`，其中`L_traj`通过软码本松弛计算轨迹损失（FK变换后与目标位置L1距离）。
- **推理细化**：采用MaskControl的细化协议（可选的静态/动态Stage-1日志优化 + Stage-2令牌嵌入优化）。

### 算法流程（文字说明）
1. **PartVQ分区**：计算每关节相对于父节点的激活幅度，标准化后计算关节间最大滞后互相关，层次聚类得到6部分，强制运动链完整性。  
2. **T-Concat骨干预训练**：在文本-运动数据上训练掩码令牌预测，冻结后用于控制适配。  
3. **控制适配**：  
   - 冻结PartVQ编码器/解码器、T-Concat骨干、CLIP文本编码器。  
   - 训练：共享轨迹编码器（1D卷积堆叠）+ 每层零初始化低秩K/V投影 + 注意力偏置标量。

## 6. 主要结论与发现

- KV-Control在继承的细化协议下达到**0.40 cm**单关节根误差（M3）和**0.71 cm**多关节误差，与MaskControl相当（0.72 cm），但机制参数仅1.5M（约MaskControl的**1/26**）。

- 纯细化（无适配器）位置误差可降至接近0，但FID飙升至~104，说明适配器提供保持运动质量的结构性条件。  
- 单关节控制中脚部FID（0.117/0.102）略高于其他关节，表明密集接触约束对适配器更具挑战。

## 7. 优点

- **参数效率极高**：仅1.5M机制参数（+9M共享轨迹编码器）实现与MaskControl相当的精度，远小于ControlNet复制分支的39.3M。  
- **透明且非破坏性**：键/值侧注入不修改查询流，保留预训练文本交叉注意力，控制与文本条件松耦合。  
- **布局-机制协同设计**：PartVQ+T-Concat+K/V注入耦合设计使每帧-部件成为可寻址注意力槽，有利于低秩控制。  
- **初始化接近恒等**：零初始化W_down和负偏置b_i使控制器开始为近似恒等，避免破坏预训练先验，可快速收敛。  
- **支持可选细化**：提供M0-M3操作点，允许用户根据精度/计算需求调整。

## 8. 不足与局限

- **数据集单一**：仅在HumanML3D上评估，未验证在其他文本-运动数据集或不同骨架下的泛化能力。

- **仅适用于离散令牌骨干**：PartVQ/T-Concat是为掩码离散令牌模型设计的，移植到连续扩散或自回归骨干需重新设计。  
- **单骨架固定**：PartVQ分区基于固定骨架，跨骨架适配需重新训练分区。
