## 摘要
生成人物交互（HOI）是角色动画、机器人、增强/虚拟现实和具身智能的核心。最近的HOI生成方法从文本、物体几何和稀疏路径点合成动作，控制动作语义和物体轨迹。然而，这些信号对交互的指定不足：相同的提示和轨迹可能产生不同的抓取、接近方向、身体姿势、物体姿势、接触和身体-物体布局。我们通过将参考图像作为所需交互快照的视觉规范来解决这种模糊性。然而，单一的全局图像表示会混淆不同的线索，并以相同的视觉证据来调节所有帧。因此，我们引入了IMAGIN-4D，一种基于扩散的HOI生成器，它在时空上分解图像条件。对于空间条件，IMAGIN-4D在描绘的帧中提取有监督的交互状态令牌，包括身体姿势、物体姿势、身体-物体接触和空间关系。对于时间条件，它通过查询每个生成帧的图像补丁来计算帧感知令牌，允许序列片段从同一图像中关注不同的视觉线索。为了平衡图像、文本和路径点线索，IMAGIN-4D使用角色感知条件：文本、路径点和交互状态令牌使用独立的AdaLN流，而帧感知视觉令牌与运动令牌进行交叉注意力。由于HOI运动数据集缺乏配对图像，我们从FullBodyManipulation（FBM）构建了一个合成运动到图像的渲染流水线，并引入了一个图像一致性度量来评估生成的运动是否与参考快照匹配。在FBM和BEHAVE上的实验表明，IMAGIN-4D在保持路径点跟随和运动质量的同时，优于单令牌和统一图像条件的基线，实现了更精细的交互控制。代码和模型将在https://imagin4d.github.io发布。

## Abstract
Generating human-object interactions (HOI) is central to character animation, robotics, AR/VR, and embodied AI. Recent HOI generation methods synthesize motion from text, object geometry, and sparse waypoints, controlling action semantics and object trajectories. However, these signals underspecify interaction: the same prompt and trajectory can produce different grasps, approach directions, body poses, object poses, contacts, and body-object layouts. We address this ambiguity with a reference image as a visual specification of the desired interaction snapshot. However, a single global image representation conflates distinct cues and conditions all frames on identical visual evidence. We therefore introduce IMAGIN-4D, a diffusion-based HOI generator that decomposes image conditioning spatio-temporally. For spatial conditioning, IMAGIN-4D extracts supervised interaction-state tokens for body pose, object pose, body-object contact, and spatial relationships at the depicted frame. For temporal conditioning, it computes frame-aware tokens by querying image patches per generated frame, allowing sequence segments to attend to different visual cues from the same image. To balance image, text, and waypoint cues, IMAGIN-4D uses role-aware conditioning: text, waypoints, and interaction-state tokens use separate AdaLN streams, while frame-aware visual tokens cross-attend with motion tokens. Since HOI motion datasets lack paired images, we build a synthetic motion-to-image rendering pipeline from FullBodyManipulation (FBM) and introduce an image-adherence metric to evaluate whether generated motions match the reference snapshot. Experiments on FBM and BEHAVE show that IMAGIN-4D improves fine-grained interaction control over single-token and uniformly image-conditioned baselines while preserving waypoint-following and motion quality. Code and models will be released at https://imagin4d.github.io.

## 1. 核心问题与整体含义
- **研究动机**：现有的人-物交互（HOI）生成方法主要依赖文本提示、物体几何和稀疏路径点来控制动作语义和物体轨迹，但这些信号对交互的指定存在严重模糊性——相同的文本和轨迹可能对应不同的抓取方式、接近方向、身体姿势、物体姿势、接触模式及身体-物体空间布局。这种歧义性阻碍了对交互细节的精确控制，尤其对于指令遵循型智能体（如机器人抓取杯柄、正确打开抽屉）至关重要。
- **整体含义**：本文提出了一种利用**参考图像**作为直观视觉规范的方法，来解决上述歧义。核心思想是将参考图像分解为**空间交互状态**（描绘帧的身体姿态、物体姿态、接触、空间关系）和**时间帧感知**（为每个运动帧从图像补丁中提取相关信息），并通过**角色感知条件机制**（文本、路径点、交互状态令牌使用独立AdaLN流，帧感知令牌使用交叉注意力）平衡多种条件信号，从而生成更精确、可控的4D人-物交互序列。

## 2. 方法论
- **核心思想**：基于扩散模型，在条件生成中引入**时空图像条件分解**和**角色感知条件机制**，使模型既能利用参考图像的细粒度空间线索（如接触点、姿态），又能根据时间位置灵活提取图像信息（如早期帧关注物体身份，晚期帧关注接触细节）。
- **关键技术细节**：
  - **空间因子化图像编码器（SFIE）**：使用冻结的DINOv2提取图像补丁，然后通过四个独立的Q-Former头部（分别对应**人体姿态、物体姿态、接触、空间关系**）学习有监督的交互状态令牌。每个令牌被训练匹配对应角色自编码器的潜在码。
  - **参考帧定位器**：由空间令牌的汇总表示预测参考图像对应的帧索引，使用高斯平滑标签监督。
  - **窗口门控空间图像条件**：对空间令牌摘要施加以预测帧为中心的高斯窗口，且在指定路径点的帧处将门控置零，避免图像条件与显式轨迹约束冲突。
  - **帧感知图像编码器（FAIE）**：每个运动帧使用帧索引和文本嵌入作为查询，重新查询同一图像补丁，生成帧依赖的视觉令牌（通过交叉注意力注入去噪器最后一层）。
  - **角色感知条件去噪器**：
    - 文本、路径点、窗口门控空间图像令牌分别通过**独立的AdaLN流**调制Transformer层的缩放、偏移和残差门控参数（图像流零初始化）。
    - 帧感知令牌通过**最后一层交叉注意力**注入运动令牌，而非全局调制。
  - **训练损失**：包含扩散损失（在预测参考帧附近加权）、SFIE监督损失、参考帧定位损失、图像一致性损失、前向运动学损失、物体点损失、脚接触损失。使用图像分类器无引导（image CFG），并在最后20步应用网格级手部和脚部引导（减少手-物体穿透和脚-地面滑动）。
- **公式和算法流程**（文字说明）：
  - 输入：文本Y、物体几何O、稀疏路径点W、参考图像I。
  - 输出：T帧运动序列 x0 ∈ R^{T×D}。
  - 图像编码：DINOv2→补丁P；SFIE生成空间令牌（接触κ、人体姿态ρ、物体姿态ξ、空间关系ν）；预测帧索引t̂；计算窗口门控空间令牌ζ̃_t；FAIE生成帧感知令牌μ_t。
  - 去噪：每个层使用AdaLN融合文本、路径点、空间图像调制；最后一层用交叉注意力加入帧感知令牌。最终输出预测清洁运动x̂0。
  - 推理时使用image CFG和网格级引导。

## 3. 实验设计
- **数据集**：
  - **FullBodyManipulation (FBM)**：9个训练物体类别，4个保留测试类别，使用CHOIS的文本提示和120帧窗口。训练和主评估使用**SceneImg**（在Replica场景中渲染带纹理的身体和物体），消融使用MeshImg（纯白背景），跨域评估使用EditImg（用FLUX.2-editing对SceneImg进行照片级编辑）。
  - **BEHAVE**：由于SMPL-H身体不兼容BEDLAM纹理，仅使用**MeshImg**训练和评估。
- **评估指标**：
  - **图像遵循度**（新指标）：衡量生成序列实现参考交互状态的程度，报告在真实帧（A^GT）、±10帧窗口（A^W10）、全序列（A^Any）下的距离（cm），越低越好。
  - **运动质量与文本对齐**：FID和R-Precision（使用冻结的CHOIS评估器）。
  - **接触**：手-物体接触的Precision/Recall/F1。
  - **交互伪影**：脚滑动、手-物体穿透。
  - **路径点误差**：生成物体轨迹与稀疏路径点的平面距离。
- **对比方法**：
  - **无图像基线**：CHOIS、MDM、InterDiff（均使用文本+物体+路径点）。
  - **图像条件基线**：CHOIS+Img（使用单池化图像token，分别用CLIP、DINOv2、Qwen2.5-VL）；ViHOI*（将ViHOI适配为单图像设置，使用Qwen2.5-VL+Q-Former）。
  - **本文变体**：Ours(Temporal)仅帧感知，Ours(Spatial)仅空间因子化，Ours(Spatial+Temporal)完整模型。

## 6. 主要结论与发现
- **图像条件是有效的交互细节规范**：参考图像可解决文本和路径点无法唯一指定的交互歧义，但**单全局token会丢失细粒度空间线索**，导致图像遵循度差（A^Any约13-13.3 cm）。
- **空间因子化令牌显著提升图像遵循度**：Ours(Spatial)将A^Any从12.56 cm降至7.95 cm，同时FID从0.52改善至0.30。
- **帧感知令牌进一步改善运动质量和接触**：加入帧感知令牌后（完整模型），FID降至0.28，接触F1提升至0.677，A^Any进一步降至7.45 cm；但路径点误差略有增加（5.69 cm，相较于CHOIS的2.87 cm），表明图像条件与路径点条件存在竞争。
- **角色感知条件是关键**：去除独立AdaLN流将A^GT从8.4 cm恶化至13.4 cm；帧感知令牌通过交叉注意力优于AdaLN（FID从0.60改善至0.28）。
- **跨域泛化需要场景级训练域**：在SceneImg上训练的模型迁移至EditImg效果较好（A^GT从8.43增至12.68 cm），而MeshImg训练域迁移至SceneImg或EditImg效果很差（A^GT >30 cm）。
- **草图条件可行但略逊于RGB**：草图条件A^W10为9.66 cm（RGB为7.65 cm），表明模型能利用形状信息。
- **模型真正使用了图像**：水平翻转图像后，生成运动的接触侧和布局随之改变，证明因果使用图像而非忽略它。

## 7. 优点
- **方法创新性**：首次将参考图像分解为**空间交互状态**和**时间帧感知**，并通过**角色感知条件**平衡多源信号，直观且有效。
- **完备的数据合成与评估**：构建了运动→图像渲染流水线，生成三种图像域（MeshImg/SceneImg/EditImg），并设计了**图像遵循度指标**（A^GT/A^W10/A^Any），系统评估条件控制精度。
- **系统性消融**：对每个设计选择（空间令牌角色、image CFG、角色条件路由、视觉模态）进行了单变量消融，清晰表明各组件贡献。
- **跨域和跨数据集验证**：在FBM和BEHAVE上均取得一致性改进，且测试了图像域迁移和草图条件，显示了鲁棒性和泛化潜力。
- **实用性**：支持草图输入，允许用户通过简单线条指定交互状态，降低使用门槛。

## 8. 不足与局限
- **路径点跟随下降**：完整模型路径点误差（5.69 cm）高于CHOIS（2.87 cm），表明图像条件与轨迹条件存在权衡。作者在消融中分析了这是“Pareto前沿”的主动选择，但未提供联合优化方案。
- **接触/穿透伪影**：尽管使用了网格级引导，手-物体穿透仍存在（0.53 cm，仅稍好于CHOIS），且对于小或薄物体尤其明显。脚滑动指标也未显著优于基线。
- **实验覆盖有限**：
  - 仅两个数据集（FBM和BEHAVE），且FBM以手部操作场景为主，未在全身复杂交互（如搬运、坐姿）或情景交互（如桌椅场景）数据集上验证。
  - BEHAVE上仅用MeshImg，未测试真实照片场景（因纹理不匹配），限制结论的普适性。
  - 所有比较基线均为CHOIS体系下的变体，未与近期基于视频/VLM的工作（如InterAct、HoD）直接比较（这些方法未使用图像条件，但可对比运动质量）。
- **图像条件帧的依赖**：方法要求参考图像提供显式交互状态，若图像与真实运动分布差异大（如EditImg域外图像、艺术风格图），性能可能大幅下降（Table 3中MeshImg→EditImg的A^GT高达30.8 cm）。
- **预测参考帧的误差**：参考帧定位由轻量MLP从空间令牌预测，定位错误会传播至窗口门控和损失加权，但论文未分析定位精度。
- **训练复杂度**：解耦的多头Q-Former和角色AdaLN流增加模型参数量和训练难度；超参数（高斯窗口宽度、损失权重）需精细调整。
- **开放性问题**：方法仅适用于单参考图像，无法处理多阶段交互（如“先推后拉”）中需要多个视觉规范的情况。纹理、照明等外观因素与交互状态纠缠，可能影响泛化。
