## 摘要
生成人物交互（HOI）是角色动画、机器人技术、AR/VR和具身智能的核心。最近的HOI生成方法从文本、物体几何和稀疏路径点合成动作，控制动作语义和物体轨迹。然而，这些信号对交互的描述不够精确：同一提示和轨迹可能产生不同的抓取、接近方向、身体姿势、物体姿势、接触和身体-物体布局。我们通过参考图像作为所需交互快照的视觉规范来解决这种模糊性。但是，单一的全局图像表示混淆了不同的线索，并使所有帧基于相同的视觉证据进行条件化。因此，我们引入了IMAGIN-4D，一种基于扩散的HOI生成器，它在时空上分解图像条件化。对于空间条件化，IMAGIN-4D提取有监督的交互状态标记，包括所描绘帧的身体姿势、物体姿势、身体-物体接触和空间关系。对于时间条件化，它通过查询每生成帧的图像块来计算帧感知标记，允许序列片段从同一图像中关注不同的视觉线索。为了平衡图像、文本和路径点线索，IMAGIN-4D使用角色感知条件化：文本、路径点和交互状态标记使用独立的AdaLN流，而帧感知视觉标记与动作标记进行交叉注意力。由于HOI运动数据集缺少配对图像，我们从FullBodyManipulation（FBM）构建了合成运动到图像的渲染管道，并引入了图像一致性度量，以评估生成的运动是否匹配参考快照。在FBM和BEHAVE上的实验表明，与单标记和统一图像条件化基线相比，IMAGIN-4D在保持路径点跟踪和运动质量的同时，改进了细粒度的交互控制。代码和模型将在https://imagin4d.github.io上发布。

## Abstract
Generating human-object interactions (HOI) is central to character animation, robotics, AR/VR, and embodied AI. Recent HOI generation methods synthesize motion from text, object geometry, and sparse waypoints, controlling action semantics and object trajectories. However, these signals underspecify interaction: the same prompt and trajectory can produce different grasps, approach directions, body poses, object poses, contacts, and body-object layouts. We address this ambiguity with a reference image as a visual specification of the desired interaction snapshot. However, a single global image representation conflates distinct cues and conditions all frames on identical visual evidence. We therefore introduce IMAGIN-4D, a diffusion-based HOI generator that decomposes image conditioning spatio-temporally. For spatial conditioning, IMAGIN-4D extracts supervised interaction-state tokens for body pose, object pose, body-object contact, and spatial relationships at the depicted frame. For temporal conditioning, it computes frame-aware tokens by querying image patches per generated frame, allowing sequence segments to attend to different visual cues from the same image. To balance image, text, and waypoint cues, IMAGIN-4D uses role-aware conditioning: text, waypoints, and interaction-state tokens use separate AdaLN streams, while frame-aware visual tokens cross-attend with motion tokens. Since HOI motion datasets lack paired images, we build a synthetic motion-to-image rendering pipeline from FullBodyManipulation (FBM) and introduce an image-adherence metric to evaluate whether generated motions match the reference snapshot. Experiments on FBM and BEHAVE show that IMAGIN-4D improves fine-grained interaction control over single-token and uniformly image-conditioned baselines while preserving waypoint-following and motion quality. Code and models will be released at https://imagin4d.github.io.

- **核心问题**：如何利用一个**参考图像**（可视为交互快照的视觉规范）来消除歧义，实现对交互细节的精细化控制，同时不降低运动质量和路径点跟随能力。
- **挑战**：直接使用单全局图像token会混淆不同交互线索（如姿态、接触、布局），并强制所有运动帧基于同一视觉证据，忽略了时间动态性；此外，现有HOI运动数据集缺少与运动配对的图像，无法直接进行图像条件训练。
- **整体含义**：IMAGIN-4D提出**时空图像条件化**和**角色感知条件化**，在保持文本和路径点控制的同时，利用参考图像提供细粒度的交互状态规范，实现了更可控的4D HOI生成。

## 2. 论文提出的方法论

### 核心思想
将图像条件分解为**空间**和**时间**两部分：空间部分通过监督式交互状态token提取参考帧的精确交互状态（人体姿态、物体姿态、接触、空间关系）；时间部分通过帧感知token为每帧从同一图像中查询不同视觉线索。同时采用**角色感知条件化**，为不同条件信号（文本、路径点、图像空间token、帧感知token）分配独立的调制路径，避免互相干扰。

### 关键技术细节

1. **条件运动公式**：
   - 采用条件扩散模型，去噪器Dθ输入带噪运动xn，预测干净运动̂x0。条件c包含文本Y、物体形状O（BPS编码）、稀疏控制xcond（初始人体/物体姿态、稀疏物体路径点、最终物体平移）。
   - 二进制掩码m标记哪些运动值是用户指定的（如路径点），哪些需要合成。

2. **空间图像条件（Spatially Factorized Image Encoder, SFIE）**：
   - 使用冻结的DINOv2提取图像块token P ∈ R^{S×C}。
   - 为每个角色h ∈ H（人体姿态ρ、物体姿态ξ、接触κ、空间关系ν）分配一个独立的Q-Former [Li et al. 2023a]，用可学习查询Q^h读取P，得到角色特定token F^h。
   - 每个token通过轻量级角色自编码器（从配对运动序列中提取目标）进行监督，损失函数LSFIE包括接触位置MSE、接触标志BCE、人体姿态MSE、物体姿态MSE、空间关系余弦距离。
   - 将所有token拼接投影为总摘要̄ζ，用于参考帧定位和图像AdaLN流。

3. **参考帧定位器**：
   - 从̄ζ预测帧分布π = softmax(W2 GELU(W1̄ζ))，用高斯平滑标签（中心为真值帧t*）监督，损失Lfp。
   - 预测帧t̂ = arg max π_k，用于训练和推理的图像窗口操作，确保去噪器在相同的定位误差下训练。

4. **窗口门控空间图像条件**：
   - 空间token摘要̄ζ通过以t̂为中心的高斯窗口w_g和路径点门控rt（若帧有显式路径点约束则rt=0，否则1）得到˜ζ_t = r_t * w_g(t - t̂) * ̄ζ。避免图像条件与路径点冲突。

5. **时间图像条件（Frame-Aware Image Encoder, FAIE）**：
   - 对每个运动帧t，用帧索引和文本嵌入作为查询Q^fvt(t, e_y)，通过Q-Former重新查询图像块P，得到帧感知token μ_t。集合M = {μ_t}。
   - 帧感知token仅通过去噪目标训练，无显式监督，学习为不同帧提供不同的图像线索（如近参考帧关注精确接触，早/晚帧关注物体身份或粗略布局）。

6. **角色感知运动去噪器（Role-Aware Denoiser）**：
   - 使用Transformer架构（4层，宽度512，4注意力头）。
   - 三种条件通过独立AdaLN流调制：文本+扩散步（base stream）、路径点（wpt stream）、窗口门控空间图像token（img stream）。每层参数η_t^(ℓ) = AdaLN_text + AdaLN_wpt + AdaLN_img，图像流零初始化。
   - 帧感知视觉token M通过最后一层解码器的交叉注意力注入：运动token作为查询，M作为键/值，仅在最后一层应用。
   - 辅助损失：正向运动学损失LFK、物体点损失Lobjpts、足部接触损失Lfeet、图像一致性损失Limg（在真值帧t*处比较解码结果）。

7. **训练与采样**：
   - 总损失L = Ldiff + LSFIE + λfp Lfp + λimg Limg + λFK LFK + λobjpts Lobjpts + λfeet Lfeet。
   - 扩散步骤1000步，余弦噪声调度，预测x0。
   - 图像分类器自由引导（图像条件丢弃概率p_drop=0.1，引导尺度s_img可调）。
   - 采样最后20步应用网格级手和脚引导以减少穿透和滑步。

8. **合成图像生成**：
   - 由于无配对图像，从FBM和BEHAVE的MoCap序列渲染图像。生成三种域：MeshImg（白背景）、SceneImg（Replica室内场景+BEDLAM纹理）、EditImg（对SceneImg用FLUX.2-dev编辑提高真实感）。
   - 参考帧选择：手部接触标志的时间质心。

## 6. 论文的主要结论与发现

1. **单token全局图像条件信息不足**：CHOIS+Img和ViHOI*虽改善FID，但图像一致性指标（AAny约13cm）远低于IMAGIN-4D（7.45cm），表明单token丢失了细粒度空间线索。
2. **空间分解token有效**：Ours (Spatial)将AAny从12.56cm降至7.95cm，证明监督式角色token能精确恢复参考交互状态。
3. **时间帧感知token进一步改善运动质量**：Ours (Spatial+Temporal)将FID从0.30降至0.28，接触F1从0.663升至0.677，说明为不同帧提供不同视觉线索有利于整体运动。
4. **角色感知条件化是关键**：移除图像AdaLN流会严重退化图像一致性（AGT由8.4升至13.4cm），而将帧感知token改为AdaLN流会大幅恶化FID（0.28→0.60）。
5. **跨域迁移显示SceneImg训练更鲁棒**：SceneImg→EditImg仅下降4.25cm，而MeshImg→EditImg下降约23cm，证明含场景纹理的训练域对真实感参考图像泛化更好。
6. **草图也可作为条件输入**：虽然图像一致性降低（AAny 9.00cm vs 7.45cm），但仍优于单token基线，展示了交互式编辑的实用性。
7. **方法对图像翻转敏感**：翻转参考图像导致抓取侧和接触区域相应改变，证明模型确实利用了图像信息而不是忽略它。

## 7. 优点（方法或实验设计亮点）

- **创新性**：首次提出**时空分解图像条件**用于HOI生成，区分空间交互状态token和时序帧感知token，解决了单token的信息压缩问题。
- **角色感知条件化**：将不同信号（文本、路径点、图像）通过独立AdaLN流和交叉注意力路由，避免条件间的竞争，是一种简单有效的设计。
- **系统性评估**：引入新指标**图像一致性（Image Adherence）**，并构建三种渲染域（Mesh/Scene/Edit）和草图测试，全面评估了图像条件生成的鲁棒性。

- **消融全面**：单独移除每个空间token、改变引导策略、更换帧感知路由等，每个设计选择都通过控制变量实验验证。

- **真实照片缺失**：所有实验条件图像均由合成渲染或编辑生成，未使用真实世界照片（如随意拍摄或网络图片），对真实域泛化能力未知。
- **BEHAVE评估简化**：仅使用MeshImg，未采用SceneImg（因纹理不兼容），也未测试跨域迁移或EditImg，限制了结论的通用性。
- **单参考图像限制**：只能指定一个交互快照，缺少对完整“先接近-然后操作-再离开”动作链的精细控制（例如抓取前的准备姿态）。模型可能在其他帧实现接触，但时间对齐不完美（见AGT vs AW10差距）。
- **物体类别有限**：FBM仅9训练+4测试类别，BEHAVE类别也较少，对复杂或非刚性物体（如衣物、折叠物）未测试。

### 方法本身的局限
- **网格穿透与物理**：虽然应用了手/脚引导，但手-物体穿透仍存在（见Tab.1 HandP指标），未完全解决物理有效性。
- **运动质量权衡**：全方法（Spatial+Temporal）的FID（0.28）优于CHOIS（0.69），但路径点误差（5.69cm）稍高于CHOIS（2.87cm），表明图像条件与路径点之间存在竞争。补充材料承认位于Pareto前沿，选择图像一致性优先，但某些应用可能更注重路径点精度。
- **参考帧定位的误差**：定位器可能预测不准，影响窗口门控和损失加权，导致生成运动偏离预期时间。实验显示AAny（全序列最佳）优于AGT（准确帧），说明时间错位普遍。
- **草图条件信息量不足**：线条画丢失外观和颜色线索，虽然推理仍可工作，但图像一致性下降较多，限制了其在需要精细接触位置的场景中的实用性。

### 偏差风险
- **合成渲染风格偏差**：SceneImg基于Replica场景和BEDLAM纹理，存在特定光照和材质模式，可能使模型学习到与真实世界照片不同的关联。
- **评估指标偏差**：FID和R-Precision使用冻结CHOIS评估器，该评估器本身可能在HOI数据集上存在偏差；图像

图像一致性指标也是基于渲染域定义的，可能无法直接迁移到真实图像评估，导致实验结果的生态效度受限。

- **偏差风险**（续）：此外，图像条件化中的Q-Former和DINOv2均在大规模通用图像集上预训练，但并未针对HOI物理精细化进行微调，因此在复杂交互场景下（如手部被物体严重遮挡）可能提取到不充分的视觉特征。空间token的监督信号完全来自配对运动序列的原始姿态和接触标签，而运动数据本身的采集误差（如标记点漂移、接触二值标注的噪声）会通过自编码器传递给条件token。

## 9. 对领域的启示与未来方向

### 对研究范式的启示

IMAGIN-4D的两个核心思想——**时空解耦**与**角色感知条件路由**——为多模态条件生成提供了通用框架。未来的工作可将其扩展到：视频/多图像条件（通过类似帧感知token的时序扩展）、音频驱动手势生成（将图像替换为音频特征）、以及更复杂的物理仿真引导（如力/力矩条件）。

### 可改进的方向

1. **增强真实感与物理一致性**：引入可微物理仿真器或接触力网络，在扩散过程中施加软约束，减少穿透和滑步；使用更高质量的3D人体/物体表示（如SMPL-X+可微渲染）。
2. **扩大数据域与物体类别**：从互联网视频或行为捕捉数据库中提取真实HOI运动-图像对（尽管困难），或利用视频生成模型（如SORA）合成大规模多样本。
3. **多参考与交互时序控制**：允许用户输入多个快照（如抓取前、抓取中、释放后），模型自动对齐时间线，实现更丰富的动作编排。
4. **交互式编辑工具**：草图条件已展现潜力，未来可结合用户笔画、区域拖拽等交互，实现实时迭代的4D内容创建。
5. **度量标准标准化**：建立包含真实世界图像域、接触物理、人体感知评估的HOI生成基准，避免对单一评估器或渲染域的过度依赖。

## 10. 总体评价

IMAGIN-4D在生成式人物交互领域取得了显著进展：**首次证明单张参考图像可以显著消除文本+路径点条件下的交互歧义**，并提出了行之有效的时空分解与角色感知路线。实验设计全面、消融严谨，为后续研究提供了高质量基线。尽管存在真实域泛化有限、物理一致性不足以及时间对齐误差等短板，但其方法创新性与系统框架的通用性使其成为该方向的重要里程碑。推荐关注HOI运动生成、多模态可控生成以及4D内容创建的学者精读。
