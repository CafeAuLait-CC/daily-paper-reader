## 摘要
文本条件的3D人体动作模型现在可以从提示语合成合理的动作，但实际的动画和具身智能体工作流程很少止步于文本：角色可能需要遵循一条草绘的根部路径，击中一个末端执行器目标，或者满足多关节轨迹，同时仍然保留语言所描述的步态、风格和意图。这暴露了一个控制权衡。一个轨迹控制器应该精确而不覆盖预训练的文本条件动作先验，但现有的解决方案要么复制生成器的大部分以重新获得每层控制访问，要么将大部分成本转移到测试时优化。我们引入了KV-Control，一个针对冻结掩码文本到动作变换器的紧凑注意力侧控制接口。关键思想是将几何约束作为记忆存储在自注意力内部，而不是通过全局姿态令牌注入或仅在输出侧强制执行。为了支持这个接口，我们共同设计了一个部分令牌化的动作基底和控制器：PartVQ学习解剖学对齐的部分码本，T-Concat将每个帧-部分令牌暴露为可寻址注意力的位置，而KV-Control在每个自注意力层注入控制条件的键/值记忆，同时保留预训练的查询流、文本交叉注意力、前馈网络和所有骨干权重。由此产生的适配器在共享的轨迹编码器之上仅添加可训练的注入参数，但在继承的细化协议下以亚厘米精度跟踪根部和多关节约束，同时保留文本条件的动作质量。KV-Control将轨迹条件化重新定义为轻量级记忆检索，为文本到动作生成提供了一个小巧、精确且透明的控制接口。

## Abstract
Text-conditioned 3D human motion models now synthesize plausible motions from prompts, but practical animation and embodied-agent workflows rarely stop at text: a character may need to follow a sketched root path, hit an end-effector target, or satisfy a multi-joint trajectory while still preserving the gait, style, and intent described by language. This exposes a control trade-off. A trajectory controller should be precise without overwriting the pretrained text-conditioned motion prior, yet existing solutions either duplicate large portions of the generator to regain per-layer control access or move much of the cost to test-time optimization. We introduce KV-Control, a compact attention-side control interface for frozen masked text-to-motion transformers. The key idea is to make geometric constraints available as memory inside self-attention rather than injecting them through a global pose token or enforcing them only at the output side. To support this interface, we co-design a part-tokenized motion substrate and controller: \textbf{PartVQ} learns anatomy-aligned part codebooks, T-Concat exposes each frame--part token as an attention-addressable site, and KV-Control injects control-conditioned key/value memories at every self-attention layer while preserving the pretrained query stream, text cross-attention, FFN, and all backbone weights. The resulting adapter adds only trainable injection parameters atop a shared trajectory encoder, yet tracks root and multi-joint constraints with sub-centimeter accuracy under the inherited refinement protocol while retaining text-conditioned motion quality. KV-Control reframes trajectory conditioning as lightweight memory retrieval, providing a small, precise, and transparent control interface for text-to-motion generation.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心矛盾**：现有轨迹控制方法面临 **精度与文本先验保持** 之间的权衡：
  - **ControlNet式重复分支**：复制生成器的大部分以获取每层自注意力访问，但可训练参数大（可达骨干的显著比例），代价高。
  - **测试时优化方法**（如TLControl、DNO）：避免参数开销，但每个样本都要额外优化，推理计算量大。
- **论文目标**：提出一种参数高效的注意力侧控制接口，在冻结的文本到运动Transformer中，通过注入控制条件的键/值记忆（K/V），实现精确轨迹跟踪且不影响文本条件运动质量。

## 2. 论文提出的方法论

### 2.1 核心思想
- 将轨迹控制重构为**自注意力内部的轻量级记忆检索**——在冻结骨干的每个自注意力层，附加控制条件的键/值对作为伪令牌，不改变原始查询流、文本交叉注意力、FFN和骨干权重。
- 需要配合**部分令牌化的运动基底**，使每个身体部分的帧-令牌成为注意力可寻址的位置，便于K/V注入精准着陆。

### 2.2 关键技术细节（三个组件）

#### （1）PartVQ：解剖学感知的部分令牌化
- 通过数据驱动的层次聚类（基于关节激活的交叉相关和运动链完整性后处理）将人体分为 Q=6 个部分。
- 对每个部分分别训练VQ-VAE码本（128个码字，码维128），编码器为1D卷积，解码器共享。
- 训练损失：smooth-L1重建 + VQ承诺 + 关节位置smooth-L1。

#### （2）T-Concat：骨干布局
- 将Q个部分码本沿序列轴展开（而非通道拼接），得到 S = T_tok × Q 个令牌，每个（帧，部分）对成为独立的注意力槽。
- 位置编码：时间编码 + 部分索引编码相加。
- 骨干为20层后归一化Transformer（d_model=384, 8头），每个自注意力层后跟随门控文本交叉注意力（使用冻结CLIP ViT-B/32）。
- 训练：掩码令牌预测 + 混合2D掩码正则化 + CFG dropout。

#### （3）KV-Control：K/V注入机制
- **控制编码器**：从当前令牌状态 z_s 经冻结PartVQ解码器得到预测位置 p_pred，结合目标 p_tgt 和残差构建控制输入 c_s（1D卷积下采样至 T_tok 个令牌）。
- **逐层注入**：每层i，将共享控制特征 f_ctrl 通过零初始化的低秩下投影（W_down_i ∈ R^{d_model×r}，r=64）和两个并行上投影（W_K_i, W_V_i ∈ R^{r×d_model}）生成控制键值 C_K_i, C_V_i（长度 T_tok）。
- 将控制键值拼接到原始键值序列后：K̃_i = [X_i; C_K_i], Ṽ_i = [X_i; C_V_i]；查询保持 X_i 不变。
- 注意力计算中加入可学习偏置 B_i：对运动令牌列为0，对控制令牌列初始化为强负数（b_i = -5），使得初始阶段控制器接近无操作。
- **训练损失**：L_KV = λ_CE L_CE + λ_traj L_traj，其中L_traj通过软码本松弛计算FK监督。
- 可选推理时细化：M0纯前馈，M1只优化logits，M2静态细化，M3动态细化（MaskControl协议）。

## 6. 论文的主要结论与发现

- **参数效率**：KV-Control仅用1.5M机制参数（10.5M全控制侧）即可在M2协议下达到0.51cm骨盆误差，而ControlNet重复分支需要39.3M参数才能达到1.24cm，参数效率提升约26倍。
- **精度**：M3协议下单关节骨盆误差0.40cm，多关节0.71cm，与MaskControl的0.72cm相当，且FID和Top-3几乎无损。
- **布局重要性**：T-Concat布局（部分令牌沿序列展开）相比C-Concat（通道拼接）在相同KV-Control下误差降低约一半（0.51 vs 1.01 cm），证明解剖可寻址性的必要性。
- **控制先验的必要性**：仅靠细化（无K/V适配器）可将位置误差降至近零，但FID崩溃至~104，说明适配器提供了保持文本条件流形的结构约束。

## 7. 优点

1. **参数高效**：机制参数仅1.5M，远少于ControlNet式的重复分支。
2. **精度高**：在多种细化协议下达到亚厘米级跟踪精度，且不牺牲文本条件质量。
3. **设计精妙**：K/V侧注入而非查询侧，避免干扰文本交叉注意力；双重近恒等初始化使适配器从无操作开始学习。
4. **布局-机制协同设计**：PartVQ+T-Concat为K/V注入提供了天然的可寻址位置，是参数效率的关键。
5. **透明性**：控制信号以显式记忆形式加入自注意力，易于理解和调试。
6. **兼容性**：可套用在MaskControl的细化协议上，允许在精度和计算之间权衡（M0-M3）。

## 8. 不足与局限

1. **骨干依赖**：PartVQ+T-Concat基座需从头预训练，迁移到其他骨干（如扩散模型、自回归模型）需要重新设计。
2. **密集约束应力**：多关节控制中足部FID偏高（左足0.117，右足0.102），且密集逐帧接触约束可能使Stage-2细化过紧，泛化受限（作者在结论中指出）。
3. **单骨架局限**：PartVQ分区针对单一骨架，难以直接适配其他形态（如动物或不同比例人体），需跨骨架适配研究。
4. **训练细节不透明**：训练时长、GPU数量、超参数调节过程未报告，可复现性打折扣。
5. **对比基线有限**：仅对比了HumanML3D单数据集，未在多个基准（如KIT-ML、BABEL）上验证泛化性。
6. **测试时细化开销**：最佳精度（M3）需要约15秒/样本（H100），虽比测试时优化方法快，但对实时应用仍可能不足。
