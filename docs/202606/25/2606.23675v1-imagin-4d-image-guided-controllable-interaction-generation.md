## 摘要
生成人-物交互（HOI）是角色动画、机器人、增强/虚拟现实和具身智能的核心。最近的HOI生成方法从文本、物体几何和稀疏路径点合成运动，控制动作语义和物体轨迹。然而，这些信号对交互的指定不充分：相同的提示和轨迹可能产生不同的抓取、接近方向、身体姿态、物体姿态、接触和身体-物体布局。我们通过参考图像作为所需交互快照的视觉规范来解决这种模糊性。然而，单一的全局图像表示混淆了不同的线索，并使所有帧都基于相同的视觉证据。因此，我们引入了IMAGIN-4D，一种基于扩散的HOI生成器，它将图像条件在时空上分解。对于空间条件，IMAGIN-4D提取所描绘帧的监督交互状态令牌，用于身体姿态、物体姿态、身体-物体接触和空间关系。对于时间条件，它通过为每个生成的帧查询图像块来计算帧感知令牌，允许序列片段关注同一图像的不同视觉线索。为了平衡图像、文本和路径点线索，IMAGIN-4D使用角色感知条件：文本、路径点和交互状态令牌使用单独的AdaLN流，而帧感知视觉令牌与运动令牌进行交叉注意力。由于HOI运动数据集缺乏配对图像，我们从FullBodyManipulation（FBM）构建了一个合成运动到图像的渲染管道，并引入图像一致性度量来评估生成的运动是否匹配参考快照。在FBM和BEHAVE上的实验表明，IMAGIN-4D在保持路径点跟随和运动质量的同时，改进了对单令牌和统一图像条件基线的细粒度交互控制。代码和模型将在https://imagin4d.github.io上发布。

## Abstract
Generating human-object interactions (HOI) is central to character animation, robotics, AR/VR, and embodied AI. Recent HOI generation methods synthesize motion from text, object geometry, and sparse waypoints, controlling action semantics and object trajectories. However, these signals underspecify interaction: the same prompt and trajectory can produce different grasps, approach directions, body poses, object poses, contacts, and body-object layouts. We address this ambiguity with a reference image as a visual specification of the desired interaction snapshot. However, a single global image representation conflates distinct cues and conditions all frames on identical visual evidence. We therefore introduce IMAGIN-4D, a diffusion-based HOI generator that decomposes image conditioning spatio-temporally. For spatial conditioning, IMAGIN-4D extracts supervised interaction-state tokens for body pose, object pose, body-object contact, and spatial relationships at the depicted frame. For temporal conditioning, it computes frame-aware tokens by querying image patches per generated frame, allowing sequence segments to attend to different visual cues from the same image. To balance image, text, and waypoint cues, IMAGIN-4D uses role-aware conditioning: text, waypoints, and interaction-state tokens use separate AdaLN streams, while frame-aware visual tokens cross-attend with motion tokens. Since HOI motion datasets lack paired images, we build a synthetic motion-to-image rendering pipeline from FullBodyManipulation (FBM) and introduce an image-adherence metric to evaluate whether generated motions match the reference snapshot. Experiments on FBM and BEHAVE show that IMAGIN-4D improves fine-grained interaction control over single-token and uniformly image-conditioned baselines while preserving waypoint-following and motion quality. Code and models will be released at https://imagin4d.github.io.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：现有的人-物交互（HOI）运动生成方法（如 CHOIS、InterDiff）主要依赖文本描述、物体几何和稀疏路径点来控制动作。然而，这些信号对交互细节（如抓取方式、接触点、身体姿态、物体朝向、身体-物体空间布局）存在严重模糊性——相同的文本和路径点可能对应多种不同的交互状态。这种模糊性限制了在动画、机器人、AR/VR 等应用中的精细控制能力。
- **动机**：为了消除这种模糊性，论文提出使用一张参考图像（如照片、渲染图、甚至草图）作为“交互快照”的视觉规范，直观地指定所需姿态、接触和布局。但直接使用单一全局图像特征（如 CLIP 或 Qwen-VL 的池化 token）会混淆多种线索，且对所有生成帧施加相同约束，无法灵活适应不同时间点的需求。
- **核心挑战**：如何从参考图像中提取细粒度的空间线索（姿态、接触、布局），并让这些线索在时间上合理地影响不同帧的运动生成，同时避免与文本、路径点等其他条件冲突。

## 2. 方法论

### 核心思想

- **时空分解图像条件**：将参考图像的条件信息分解为两部分：
  - **空间条件**：通过一个“空间因子化图像编码器”（SFIE）提取多个角色特定的监督 token，分别对应人体姿态（𝝆）、物体姿态（𝝃）、接触（𝜿）和身体-物体空间关系（𝝂）。这些 token 通过轻量级角色自编码器与真实运动帧中的对应量进行监督学习。
  - **时间条件**：通过“帧感知图像编码器”（FAIE）为每个运动帧生成独立的视觉 token（𝝁_t），这些 token 基于帧索引和文本嵌入查询图像块，允许不同帧关注图像中不同的区域（例如抓取帧关注手部接触细节，而远离帧关注物体大致位置）。

### 关键技术细节

- **空间因子化图像编码器**：使用冻结的 DINOv2 提取图像块特征，然后对每种角色使用独立的 Q-Former 头，通过可学习查询向量读取块特征，输出角色 token。这些 token 被送入对应的解码器，预测接触位置/标志、人体关节、物体旋转/平移、关节到物体的单位向量，并通过均方误差或二值交叉熵损失进行监督。
- **帧感知图像编码器**：类似地使用 Q-Former，但查询向量由帧索引和文本嵌入条件生成，输出每帧的视觉 token。这些 token 通过最后一层 Transformer 的交叉注意力注入到运动解码器中，不进行直接监督，仅通过去噪损失学习。
- **角色感知条件融合**：为防止不同条件信号相互干扰，IMAGIN-4D 采用分离的 AdaLN 流（分别对应文本+扩散步、路径点约束、窗口门控的空间图像 token），而帧感知 token 则通过交叉注意力注入。空间 token 还经过窗口门控（以预测的参考帧为中心的高斯窗口），并屏蔽有指定路径点的帧，避免与显式轨迹约束冲突。
- **参考帧定位器**：从空间 token 的拼接特征预测参考图像对应的时间帧（ˆt），使用高斯平滑标签训练。定位器在训练和推理时都使用预测结果，使去噪器适应定位误差。
- **训练损失**：包括去噪损失（在参考帧附近加权重）、SFIE 监督损失、参考帧定位损失、图像一致性损失（在真实参考帧比较生成帧与图像目标的姿态/接触）、前向运动学损失、物体点损失、足部接触损失。

### 算法流程（文字说明）

1. 给定文本、物体几何、稀疏路径点、参考图像。
2. 冻结 DINOv2 提取图像块特征。
3. SFIE 使用四个 Q-Former 头提取角色 token，预测参考帧的接触、人体姿态、物体姿态、空间关系，并定位参考帧索引。
4. FAIE 为每个运动帧生成帧感知视觉 token。
5. 去噪 Transformer 接收这些条件：文本+扩散步、路径点、窗口门控空间 token 分别通过 AdaLN 流；帧感知 token 通过最后一层交叉注意力；物体几何使用 BPS 编码。
6. 通过扩散过程生成 120 帧的 HOI 运动序列。
7. 采样时还可使用图像分类器自由引导和网格级手/足引导（最后 20 步）。

## 6. 主要结论与发现

- 单一全局图像 token 虽然可以改善全局指标（FID、R-Precision），但对图像一致性的控制非常有限（A_Any 约 13 cm）。
- 空间因子化 token 显著提升图像一致性（A_Any 降至 ~8 cm），但全局运动质量下降（FID 增加）。
- 添加帧感知 token 进一步改善图像一致性（A_Any 降至 7.45 cm），同时提升 FID 和接触质量，实现最佳平衡。
- 角色感知条件（分离 AdaLN + 交叉注意力）相比混合条件，图像一致性提升 5 cm 以上，虽然路径点误差略有增加（存在固有 trade-off）。
- 图像 CFG 在采样时对图像一致性有 1 cm 左右的增益。
- 训练场景（SceneImg）比简单渲染（MeshImg）具有更好的跨域泛化能力，迁移到编辑图像（EditImg）损失可控。
- 草图输入也能提供一定的交互控制（A_W10 ~97 cm），但弱于 RGB（~77 cm），可作为可编辑输入形式。
- 参考帧定位器能够有效预测图像对应的帧（消融中未见单独定位精度分析，但模型整体合理）。

## 7. 优点

- **方法论新颖性**：首次将参考图像条件分解为空间（角色特定、监督）和时间（帧感知、自监督）两个维度，突破了单一全局 token 的信息瓶颈。
- **条件融合设计巧妙**：角色感知条件通过分离的 AdaLN 流和交叉注意力分别处理不同类型的线索，有效避免了信号冲突，这是实现精细控制的关键。
- **实验覆盖面广**：在 FBM 和 BEHAVE 两个数据集、多个图像域、多种基线（包括最新并发工作 ViHOI）上验证，消融实验系统全面。
- **实际应用价值**：支持草图输入，为用户提供了一种直观的交互控制界面；跨域泛化实验表明方法对图像风格具有一定鲁棒性。
- **开源和可复现性**：承诺发布代码和模型，提供合成渲染管道和图像一致性指标，便于后续研究。

## 8. 不足与局限

- **图像一致性指标依赖渲染假设**：作者自行定义了加权关节-物体距离作为图像一致性度量，该度量基于渲染图像与真实运动的对齐，但未必完全反映人眼感知的“一致性”。未与人类评估进行相关性验证。
- **跨域泛化仍有差距**：MeshImg 训练迁移到场景图或编辑图时性能下降显著（A_GT 从 ~8 cm 恶化到 ~31 cm），说明模型对训练域外的图像风格敏感，泛化能力有限。
- **手部穿透等伪影未完全消除**：虽然使用了网格级引导，但手-物体穿透指标在部分配置下仍较高（如 HandP 在 0.53-0.62 之间），可能影响实际应用中的视觉质量。
- **路径点跟随 trade-off**：引入图像条件后路径点误差略有增加（CHOIS 2.87 → 5.69 cm），作者承认存在竞争，但未提供完全消除矛盾的解决方案（如自适应加权机制）。
- **参考帧定位器误差**：消融实验显示移除参考帧定位器或使用不准确的 ˆt 会导致性能下降，但论文未深入分析定位精度受限的影响范围。
- **依赖渲染数据**：训练必须使用合成图像，而真实场景照片的适用性未直接验证（仅通过 EditImg 评估，但 EditImg 仍基于渲染基础）。
- **未报告计算成本**：如训练时长、推理速度等实际部署关心的指标缺失。
- **对极端交互状态可能失效**：如物体过小、接触面积极小、或参考图像中交互处于快速变化时刻，模型可能难以准确恢复。
