## 摘要
离散流匹配为离散结构上的生成建模提供了一个灵活的框架。然而，标准分解训练目标使模型面临不同难度的目标，将条件良好、可预测的标记与模糊、高熵的标记混合在一起。我们通过实验证明，每个标记值的不确定性与其邻域中可用上下文的密度密切相关。受此观察启发，我们对基础连续时间马尔可夫链（CTMC）提出了一种简单的修改，以融入局部上下文信息。我们的上下文加权采样器以可忽略的计算开销提高了生成质量，而我们的缩放交叉熵损失函数重新加权了来自不同标记的训练信号，并在OpenWebText上将生成困惑度降低了高达63%。此外，我们的方法在质量上与强半自回归块扩散基线相匹配，同时保留了以任意顺序进行生成的能力。这些结果突显了局部上下文作为离散生成建模中重要因素的作用，并表明简单的上下文感知修改可以显著提高采样和训练效率。

## Abstract
Discrete flow matching provides a flexible framework for generative modeling on discrete structures. However, the standard factorized training objective exposes the model to targets of varying difficulty, mixing well-conditioned, predictable tokens with ambiguous, high-entropy ones. We empirically demonstrate that the uncertainty over the value of each token is closely related to the density of available context in its neighborhood. Motivated by this observation, we propose a simple modification to the underlying continuous-time Markov chain (CTMC) that incorporates local context information. Our context-weighted sampler improves generation quality with negligible computational overhead, while our scaled cross-entropy loss function reweights the training signal from different tokens and reduces generative perplexity by up to 63% on OpenWebText. Moreover, our approach matches a strong semi-autoregressive block diffusion baseline in quality while retaining the ability to perform generation in any order. These results highlight the role of local context as an important factor in discrete generative modeling and show that simple context-aware modifications can significantly improve both sampling and training efficiency.

- **问题**：标准离散流匹配（DFM）中，训练目标对每个 token 位置给予同等权重，但不同 token 的预测难度差异很大：有些 token 有丰富的局部上下文（如周围已揭露 token），预测难度低、熵值小；有些 token 则模糊、高熵。这种难度不均导致模型被多类目标“混合”训练，学习信号效率低下。
- **整体含义**：作者通过实证发现 token 的预测不确定性与**局部上下文的密度**（即该 token 邻域内已揭露 token 的数量）密切相关。基于此，提出**上下文加权离散流匹配（Context-weighted Discrete Flow Matching）**，通过修改 CTMC 的转移动力学和损失函数，将局部上下文信息融入生成与训练过程，在几乎不增加计算开销的前提下显著提升生成质量，且保持“任意顺序生成”的灵活性。

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：用局部上下文（邻域内已揭露 token 数量）作为预测难度的廉价代理，据此调整 CTMC 中每个 token 位置的更新速率，并在训练时用相应的权重重新加权交叉熵损失。
- **关键技术细节**：
  1. **上下文权重函数**：定义 αᵢ(z)（依赖于当前状态 z，与目标值无关），修改因子化速度：  
     `vᵢᵗ(xᵢ, z) = αᵢ(z) · uᵢᵗ(xᵢ, z)`。  
     要求 α 在已遮蔽位置上平均为 1，且满足循环移位等变性，以保证路径边界条件和每个 token 的边际分布不变（定理 1）。
  2. **推断时采样（上下文加权采样器）**：  
     - **Neighbor-weighted 求解器**：权重 ∝ 该 token 在半径 r 的局部窗口内已揭露 token 的数量。  
     - **Entropy-weighted 求解器**：权重 ∝ 模型预测熵的倒数（低熵权重更大）。  
     使用 softmax 归一化（逆温度 s），满足定理 1 条件，可直接用于预训练模型，无需微调。
  3. **训练时路径采样**：算法 1：先采样总揭露 token 数 m ~ Binomial(N, κₜ)，然后按权重 αᵢ 逐个选择已遮蔽位置进行揭露，O(N) 复杂度返回精确的条件样本。
  4. **缩放交叉熵损失（SCE）**：将标准交叉熵按权重 αᵢ(Xₜ) 重新加权：  
     `L_α(θ) = -E[ Σ αᵢ(Xₜ) log p_θ(Xᵢ¹|Xₜ) ]`。  
     该损失强调低不确定性、可减的模型误差部分，弱化高不确定性（噪声）部分。

### 8. 不足与局限

- **低 NFE 时效果有限**：Neighbor-weighted 求解器在很少更新步数时（如 NFE=128 以下）效果弱于 Euler，原因在于邻居 token 更新过于独立，局部不一致。
- **实验范围有限**：仅使用约 170M 参数的模型和一维离散序列（文本、分子 SMILES），未验证更大模型、更长上下文或图像等二维/三维数据上的表现。
- **Entropy-weighted 求解器不保证边际分布保持**：模型预测的熵不一定满足循环移位等变性，因此不能像 Neighbor-weighted 那样有理论保证的边际分布不变性。
- **测试困惑度 vs 生成质量的不一致**：SCE 模型在标准测试集上困惑度可能更高，但生成质量更好，说明标准困惑度未必反映生成性能，需要更适配的评估方式。
- **上下文代理简单**：仅使用固定窗口内揭露邻居数量，未探索更精细的不确定性建模或自适应上下文聚合。
- **应用风险**：强生成能力可能被误用于生成虚假文本，但论文为方法论研究，未部署系统。
