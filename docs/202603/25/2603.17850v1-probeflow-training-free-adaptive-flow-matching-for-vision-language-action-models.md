## 摘要
最近，配备流匹配（Flow Matching, FM）动作头的视觉-语言-动作（VLA）模型在复杂的机器人操控中取得了最先进的性能。然而，FM 所需的多步迭代 ODE 求解引入了推理延迟，阻碍了响应式的物理控制。虽然目前的加速工作主要优化视觉-语言模型（VLM）主干网络，但动作头的瓶颈仍被忽视。为此，我们提出了 ProbeFlow，这是一种专为连续机器人控制量身定制的无需训练的自适应推理框架。通过利用初始速度向量与前瞻速度向量之间的余弦相似度来评估几何轨迹复杂度，ProbeFlow 动态调度积分步数以剪枝冗余的网络评估。在 MetaWorld 基准测试中，它将动作解码加速了 14.8 倍（平均步数从 N = 50 降至 2.6），并在不影响操控成功率的情况下将端到端系统延迟降低了 2.8 倍。在长时程 LIBERO 基准测试中，探针自动分配更密集的调度以渡过语义瓶颈，有效解决了流求解器的延迟。真实世界的物理部署证实，ProbeFlow 在确保执行稳定性的同时成功缓解了动作解码延迟，为低延迟连续生成策略提供了一种高度实用的解决方案。

## Abstract
Recent Vision-Language-Action (VLA) models equipped with Flow Matching (FM) action heads achieve state-of-the-art performance in complex robot manipulation. However, the multi-step iterative ODE solving required by FM introduces inference latency that precludes responsive physical control. While current acceleration efforts optimize the Vision-Language Model (VLM) backbone, the action head bottleneck remains overlooked. To address this, we propose ProbeFlow, a training-free adaptive inference framework tai- lored for continuous robotic control. By evaluating geometric trajectory complexity via the cosine similarity between initial and lookahead velocity vectors, ProbeFlow dynamically sched- ules integration steps to prune redundant network evaluations. On the MetaWorld benchmark, it accelerates action decoding by 14.8x (reducing average steps from N = 50 to 2.6) and cuts end-to-end system latency by 2.8x without compromising the manipulation success rate. On the long-horizon LIBERO benchmark, the probe automatically allocates a denser schedule to navigate semantic bottlenecks, effectively resolving the flow solver delay. Real-world physical deployments confirm that ProbeFlow successfully mitigates action decoding latency while ensuring execution stability, offering a highly practical solution for low-latency continuous generative policies.

*   **核心问题**：现代 VLA 模型（如 $\pi_0$、OpenVLA）常采用流匹配（FM）作为动作生成头，虽然性能优异，但其推理过程依赖于多步迭代求解常微分方程（ODE）。
*   **研究背景**：标准的固定步数求解器（如 Euler 法，通常需 20-50 步）会导致严重的推理延迟，无法满足机器人实时、高频控制的需求。
*   **动机**：作者观察到 FM 生成的概率路径并非处处弯曲，在简单的平移运动中路径接近直线，而在复杂的交互（如抓取）中路径较弯。现有的加速研究多集中在 VLM 骨干网络，而忽略了动作头这一计算瓶颈。

### 2. 论文提出的方法论
*   **核心思想**：提出一种**无需训练（Training-free）**的自适应步数调度策略。通过一个轻量级的“前瞻探针”来评估当前轨迹的几何复杂度（线性度），从而动态决定推理步数。
*   **关键技术细节**：
    1.  **前瞻线性探针（Lookahead Linearity Probe）**：在起始点 $t=0$ 计算初始速度 $v_{start}$，然后尝试迈出一个较大的“前瞻步”（如 $\Delta t=0.5$）到达预测状态 $x_{probe}$，并计算该点的速度 $v_{probe}$。
    2.  **余弦相似度评估**：计算 $v_{start}$ 与 $v_{probe}$ 之间的余弦相似度 $S$。若 $S \approx 1$，说明路径非常直，可以跳过中间步骤；若 $S \ll 1$，说明路径弯曲，需要加密采样。
    3.  **动态步数调度**：根据 $S$ 将步数 $N$ 映射到预设范围 $[N_{min}, N_{max}]$。
    4.  **状态复用**：在计算探针时产生的速度向量会被直接复用于后续积分，确保探针本身的计算开销几乎为零。

### 6. 主要结论与发现
*   **显著加速**：在 MetaWorld 上，动作头推理速度提升了 **14.8 倍**（平均步数从 50 降至 2.6），端到端系统延迟降低了 **2.8 倍**。
*   **性能保持**：在大幅加速的同时，操控成功率与 50 步的基准模型持平（甚至略有提升，因减少了累积误差）。
*   **动态分配规律**：实验证实，ProbeFlow 在物体接近和抓取等关键阶段会自动分配更多步数，而在空载移动阶段则使用极少步数。
*   **真机验证**：在真实物理环境下，ProbeFlow 成功将动作解码延迟从 270ms 降低至 12ms，确保了控制的实时性。

### 7. 优点
*   **即插即用**：无需对原模型进行任何微调或重新训练，部署成本极低。
*   **计算高效**：通过复用探针计算结果，避免了传统自适应求解器（如 RK45）因多次函数评估（NFE）导致的额外开销。
*   **几何直观**：利用 FM 算法本身“路径趋向直线”的数学特性，逻辑严密且易于理解。

### 8. 不足与局限
*   **阈值敏感性**：虽然无需训练，但线性阈值 $\epsilon$ 在不同领域（如从 MetaWorld 到 LIBERO）可能需要手动微调以获得最佳平衡。
*   **极端场景挑战**：对于动力学极其复杂、接触频繁且非线性极强的任务，固定的前瞻步长 $\Delta t_{probe}$ 可能无法捕捉到瞬时的路径变化。
*   **应用范围**：目前仅验证了基于 Flow Matching 的模型，对于传统的扩散模型（Diffusion Policy）或其他非 ODE 结构的策略是否适用尚待研究。
